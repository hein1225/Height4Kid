# 身高成长小助手（Height4Kid）使用文档

## 项目简介

身高成长小助手（Height4Kid）是一个用于记录和分析小孩身高成长的项目，分为服务端和安卓客户端两部分。服务端通过 Zeabur 部署，客户端通过 Flutter 开发。

## 系统功能

### 管理员功能

1. **账户管理**：管理管理员账号及普通账户，包括创建、删除和修改。
2. **问题反馈**：查看和回复普通用户反馈的问题。
3. **项目标题修改**：修改项目标题。
4. **国标数据导入**：更新最新的小孩身高与体重的国标数据。
5. **运行日志**：查看所有日志，分类为常规日志、错误日志。
6. **普通账户登录记录**：查看普通账户登录情况。
7. **备份与还原**：备份和还原项目数据。
8. **重置所有数据**：重置所有数据，恢复到项目初始化。

### 普通用户功能

1. **小孩信息管理**：管理小孩信息，包括添加、编辑、删除。
2. **成长信息记录**：记录小孩的身高和体重。
3. **成长曲线**：查看小孩的成长记录和曲线。
4. **问题反馈**：提交问题反馈给管理员。
5. **备份与还原**：备份和还原当前账号的用户数据。

## 管理员使用指南

### 登录

1. 打开服务端网址，例如 `https://height4kid.zeabur.app`。
2. 输入默认管理员账号和密码：
   - 用户名：admin
   - 密码：admin123
3. 点击登录按钮。

### 账户管理

1. 在管理员后台首页，点击 "账户管理" 卡片。
2. 在账户管理页面，可以：
   - 查看所有用户列表
   - 点击 "添加用户" 按钮添加新用户
   - 点击 "编辑" 按钮修改用户信息
   - 点击 "删除" 按钮删除用户

### 问题反馈

1. 在管理员后台首页，点击 "问题反馈" 卡片。
2. 在问题反馈页面，可以：
   - 查看所有用户反馈
   - 点击 "回复" 按钮回复反馈
   - 点击 "删除" 按钮删除反馈

### 国标数据导入

1. 在管理员后台首页，点击 "国标数据导入" 卡片。
2. 在国标数据导入页面，可以：
   - 查看现有的国标数据
   - 填写新的国标数据并点击 "添加数据" 按钮

## 普通用户使用指南

### 首次登录

1. 安装并打开客户端应用。
2. 输入服务端网址，例如 `https://height4kid.zeabur.app`。
3. 输入用户名和密码。
4. 点击登录按钮。

### 小孩信息管理

1. 在首页，点击 "小孩信息管理" 卡片。
2. 在小孩信息管理页面，可以：
   - 查看所有小孩列表
   - 点击 "添加" 按钮添加新小孩
   - 点击 "编辑" 按钮修改小孩信息
   - 点击 "删除" 按钮删除小孩（会同时删除该小孩的所有成长记录）

### 成长信息记录

1. 在首页，点击 "成长信息记录" 卡片。
2. 选择要记录的小孩。
3. 填写记录日期、身高和体重。
4. 点击 "保存记录" 按钮。

### 成长曲线

1. 在首页，点击 "成长曲线" 卡片。
2. 选择要查看的小孩。
3. 查看该小孩的成长记录列表。

### 问题反馈

1. 在首页，点击 "问题反馈" 卡片。
2. 填写问题标题和内容。
3. 点击 "提交反馈" 按钮。

## 技术栈

- **服务端**：Express.js (Node.js) + SQLite + React + Tailwind CSS
- **客户端**：Flutter

## 部署说明

### Zeabur 部署

#### 需要上传的文件夹

- **仅需上传 `server` 文件夹**到 Zeabur，客户端无需部署到 Zeabur。

#### 环境变量设置

在 Zeabur 控制台中，需要设置以下环境变量：

| 环境变量名 | 值 | 说明 |
|-----------|-----|------|
| `PORT` | `8000` | 服务端端口 |
| `JWT_SECRET` | `your-secret-key` | JWT 签名密钥，建议使用随机生成的字符串 |

#### 网络设置

- **端口**：确保 Zeabur 服务的端口映射为 8000。
- **域名**：Zeabur 会自动分配一个域名，例如 `height4kid.zeabur.app`。
- **HTTPS**：Zeabur 会自动配置 HTTPS，无需额外设置。

#### 硬盘设置

- **存储**：由于使用 SQLite 数据库，需要确保 Zeabur 服务有足够的存储空间。
- **持久化**：Zeabur 的文件系统是持久化的，数据库文件会保存在服务实例中。
- **备份**：建议定期通过管理员后台的 "备份与还原" 功能下载数据库备份。

### 客户端构建与签名

客户端构建与签名的详细教程请参考 `client/README.md` 文件。

## 注意事项

1. 首次登录管理员账号后，建议修改默认密码。
2. 定期备份项目数据，以防数据丢失。
3. 客户端首次登录时需要输入服务端网址，请确保输入正确的网址。
4. 国标数据需要根据实际情况定期更新。
